# DEVLOG

## 当前状态

### 已完成

- 完成动态文档站基础抓取内核
- 完成基于 `Playwright` 的页面抓取与截图保存
- 完成基于站点配置的标题、正文和噪音区域提取
- 完成 `jjjshop` 文档站左侧菜单批量抓取
- 完成批量目录索引文档自动生成
- 完成按 `category_id` 隔离批量输出目录，避免覆盖
- 完成 `analyze` 命令，支持新站点页面结构分析与配置草稿生成
- 完成 `CRMEB` 页面分析与专用配置草稿落地

### 当前可用命令

- `pachong crawl`
- `pachong batch`
- `pachong analyze`

## 已实现的关键设计

- 将“站点适配”从代码中抽离为配置文件
- 通过 `wait_selector + wait_for_text_selector` 控制页面抓取时机
- 正文优先使用 HTML 转 Markdown，减少图片型内容缺失
- 批量抓取自动输出目录文档，便于查阅
- `analyze` 用启发式规则生成站点配置草稿，降低新站接入成本

## 当前边界

- `batch` 目前仍以 `jjjshop` 站点适配为主
- `CRMEB` 已能单页分析与抓取，但批量抓取尚未接入
- `analyze` 目前能识别标题、正文、菜单和噪音区域，但还不能自动判断是否应启动批量抓取
- 还没有统一的任务状态存储、断点续跑和失败重试体系
- 还没有对接口抓取、登录态录制和代理池做系统化封装

## 下一步继续做的方向

### 1. 增强 analyze

- 增加页面分类能力：
  - `detail_page`
  - `index_page`
  - `hybrid_page`
- 增加子链接提取能力
- 增加“是否建议启动 batch”的判断逻辑

### 2. 扩展 batch

- 为 `CRMEB` 增加目录页和子页面批量抓取能力
- 逐步从站点专用批量逻辑演进为“站点适配器 + 通用 batch”

### 3. 增加任务系统

- 任务状态记录
- 失败重试
- 断点续跑
- SQLite 元数据存储

### 4. 增加接口抓取能力

- 分析 XHR / fetch 请求
- 在适合的站点上直接抓接口数据
- 将接口抓取与页面抓取结合
