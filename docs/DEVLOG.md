### 当前可用命令

- `pachong crawl`
- `pachong batch`
- `pachong analyze`

## 已实现的关键设计

- 将“站点适配”从代码中抽离为配置文件
- 通过 `wait_selector + wait_for_text_selector` 控制页面抓取时机
- 正文优先使用 HTML 转 Markdown，减少图片型内容缺失
- 批量抓取自动输出目录文档，便于查阅
- `analyze` 用启发式规则生成站点配置草稿，降低新站接入成本
- `analyze` 已能输出页面类型、候选子链接和是否建议 batch 的判断结果

## 当前边界

- `batch` 已支持根据 analyze 结果在 click_batch 和 link_batch 之间自动分流
- `link_batch` 已支持目录页候选子链接批量抓取，但仍需继续优化摘要和目录质量
- `analyze` 已能做初步 batch 候选判断，但仍需继续降低误判率
- 还没有统一的任务状态存储、断点续跑和失败重试体系
- 还没有对接口抓取、登录态录制和代理池做系统化封装

## 下一步继续做的方向

### 1. 增强 analyze

- 继续优化页面分类和 batch 评分模型
